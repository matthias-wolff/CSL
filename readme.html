1 Stable Version

To start the stable version connect a network drive L: to \\141.43.71.51\Software and execute L:\CogntiveSystemsLab-stable\CogntiveSystemsLab\CogntiveSystemsLab.bat. This will start the CogntiveSystemsLab server on FEHMARN and a client on any other workstation. The server should be started before the clients.

To suppress the security warning that occurs when double-clicking the batch script open "Control Panel/Internet Options/Security/Local Intranet/Sites/Advanced" and add "L:" to the list of intranet sites.

  1. Check out the following SVN repositories:
    Name Description URL

2.3 Speech Dialog

Modifying the voice control dialog requires dLabPro and UASR. Check-out and build

Name Description URL
UASR Unified Approach to Speech Synthesis and Recognition svn+ssh://eakss1.et.tu-dresden.de/uasr[/trunk]
dLabPro Signal processing and pattern recognition toolbox svn+ssh://eakss1.et.tu-dresden.de/dLabPro[/trunk]
dLabPro-recognizer Speech and acoustic pattern recognizer svn+ssh://eakss1.et.tu-dresden.de/dLabPro-recognizer[/trunk]
and modify the environment variables as follows:
Variable Modification
UASR_HOME Set to: <UASR-dir>
MACHINE Set to: x86-w32
PATH Add to path: <dLabPro-dir>\%MACHINE%

For further instructions see "Getting started" in the dLabPro documentation.

The CogntiveSystemsLab voice control dialog is defined in the file src/de/tucottbus/kt/csl/resources/de/csl.dlg.txt. After modifying the dialog file you need to pack the recognizer data files either

  1. by running the Ant script PackRecognizerData.xml or
  2. by running the UASR script 
      cd <CogntiveSystemsLab-dir>
  REC_PACKDATA.xtp dlg PackRecognizerData.cfg

To save compilation time you can add the command line option -DUASR.skip=FG to the Ant script or -Pskip=FG to the UASR script, respectively. This will just re-compile the dialog without re-creating any other recognizer data files.

1) Set the path to <dLabPro-dir>/bin.release.x86-w32 if you checked out the dLabPro and dLabPro-recognizer projects.
